Exploratory Analysis of the 2025 Alaska Salary Study¶

This notebook performs an exploratory analysis of the data published by the Alaska Department of Administration in their 2025 Salary Study conducted by Segal. We began by loading the raw Segal data and identifying, for each position and step, the pay range closest (in absolute difference) to the 65th-percentile market target—without simply rounding up. From there, we computed the range change required to move each position up or down to that target.

Next, we merged in headcount figures to assess the budgetary and workforce impact of those adjustments, and calculated weighted averages so our findings better reflect the distribution of employees across ranges. Throughout, we remain mindful that the State’s own analysis introduced a 50th-percentile comparison—an approach that neither reflects the long-standing interpretation of “competitive” nor aligns with legislative intent to target the 65th percentile.

# How much are we talking about in actual salary, what are we missing?

# Define categories and calculate weighted median salary increase needed for each category
categories = {
    'needs_at_least_two_range_increase': lambda x: x['range_change'] >= 2,
    'needs_at_least_three_range_increase': lambda x: x['range_change'] >= 3,
    'needs_at_least_four_range_increase': lambda x: x['range_change'] >= 4,
}

for category, condition in categories.items():
    df_category = df_p65_filtered[condition(df_p65_filtered)]
    
    if not df_category.empty:
        percentage = (df_category['headcount'].sum() / df_p65_filtered['headcount'].sum()) * 100
        employees = df_category['headcount'].sum()
        
        # Calculate weighted median salary increase needed
        df_category.loc[:, 'salary_increase_percentage'] = ((df_category['market_target_annual_salary'] - df_category['ak_annual_salary']) / df_category['ak_annual_salary']) * 100
        weighted_median_percentage = (df_category['headcount'] * df_category['salary_increase_percentage']).sum() / df_category['headcount'].sum()
        weighted_median_dollars = ((df_category['market_target_annual_salary'] - df_category['ak_annual_salary']) * df_category['headcount']).sum() / df_category['headcount'].sum()
        
        print(f"Category: {category.replace('_', ' ').title()}")
        print(f"Percentage of workforce: {percentage:.2f}%")
        print(f"Number of studied employees: {employees}")
        print(f"Weighted median salary increase needed (percentage): {weighted_median_percentage:.2f}%")
        print(f"Weighted median salary increase needed (dollars): ${weighted_median_dollars:,.2f}")
        print("-" * 50)
    else:
        print(f"No employees in category: {category.replace('_', ' ').title()}")
        print("-" * 50)

print(f"P-value = {p:.3f}")
if p < alpha:
    sig_text = "statistically significant"
else:
    sig_text = "not statistically significant"

# Categorize strength
abs_r = abs(r)
if abs_r < 0.1:
    strength = "negligible"
elif abs_r < 0.3:
    strength = "weak"
elif abs_r < 0.5:
    strength = "moderate"
else:
    strength = "strong"

direction = "negative" if r < 0 else "positive"

print(
    f"At α = {alpha}, this correlation is {sig_text}. "
    f"The effect size ({strength}, {direction}) suggests "
    f"that there is {('essentially no' if strength=='negligible' else strength)} linear association "
    f"between original range and range change."
)

Ahh, yes, that makes sense, regression to the mean...¶

In addition to what appears to be random variation, what you’re seeing is exactly the classic “regression‐to‐the‐mean” artifact:

  • Low original ranges only have room up, so they tend to show larger positive “range change.”
  • High original ranges can’t go much higher (or at all), so they often show small increases—or even decreases—to pull them back toward that 65th‐percentile “target.”

Thus, even when we plot the trend line, we're missing some of the randomness that is apparent in the scatter chart in a more traditional line chart of the mean as displayed below.
